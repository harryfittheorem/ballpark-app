What we're building:
- Edge Function: supabase/functions/mux-webhook/index.ts
- Receives webhooks from Mux for events: video.asset.ready,
  video.asset.errored, video.upload.asset_created
- Updates videos table:
  - On video.asset.ready: status='ready', sets duration_seconds,
    sets mux_playback_id
  - On video.asset.errored: status='errored'
  - On video.upload.asset_created: links upload to asset_id
- Webhook signature verification using MUX_WEBHOOK_SECRET
- Idempotent (Mux retries — UNIQUE constraints on
  mux_asset_id/mux_playback_id from Step 4.1 help here)

Mux webhook setup:
- After deploying, the function has a public URL
- Configure that URL in Mux Dashboard → Webhooks
- Mux signs webhooks with a secret you set there
- Add MUX_WEBHOOK_SECRET to Replit Secrets

Out of scope:
- Mobile client polling for video status (Step 4.14)
- Email notification on video ready (Step 4.15)
- Handling all Mux webhook events (just the three above)
