What we're building:
- Edge Function: supabase/functions/mux-create-upload/index.ts
- Auth: requires authenticated user with app_role='coach'
- Calls Mux API to create a Direct Upload URL
- Inserts a row in public.videos with status='uploading' and
  the returned mux_asset_id
- Returns to client: { upload_url, video_id }
- The client uploads bytes directly to Mux (separate step in 4.6)

Why a server-side function instead of client-direct:
- MUX_TOKEN_SECRET cannot be exposed to mobile clients
- Centralizes Mux API auth
- Server-side INSERT into videos table (with service_role) keeps
  RLS sane

Out of scope:
- Mux webhook handler (Step 4.5)
- Mobile client upload code (Step 4.6)
- Coach UI (Step 4.7+)
